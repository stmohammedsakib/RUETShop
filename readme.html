Overview

What is RUET Online Shop?

RUET Online Shop is a fully client-side campus marketplace — no servers, no npm, no build pipeline. All data is persisted in the browser via localStorage through a custom MongoDB-style DB abstraction layer. The app ships with three distinct user roles — Buyer, Vendor, and Admin — each with dedicated dashboards and scoped permissions.

Open index.html in any browser. That's it.

Features

Everything you need

🛍️

Shopping Catalog

Grid/list view, full-text search, filter by category, price, rating & stock. Sort by price, rating or newest. Paginated 12 per page.

Cart & Checkout

Per-user persisted cart, guest cart merge, real-time stock validation, bKash/Nagad/COD payment, multi-step checkout flow.

Register as Buyer or Vendor, guest login, 3-step forgot password, role-based redirects, and password visibility toggle.

Wishlist

Add, remove, and move items to cart individually or all at once. Live badge count synced across the entire UI.

📊

User Dashboard

Stats overview, full order history, wishlist panel, profile editor. Vendors get full product CRUD with modal forms.

⭐

Reviews & Ratings

Interactive star picker with hover preview. One review per user per product. Averages auto-recalculate on submission.

🛡️

Admin Panel

Platform-wide stats, user/role management, product moderation, order status updates, and one-click data reset.

🎨

UI / UX Polish

Light/dark theme, responsive layout, skeleton screens, toast notifications, animated hero counter, and CSS design tokens.

Architecture

How it works

Data Layer — DB

A singleton IIFE wrapping localStorage with a MongoDB-style collection API. Every module reads and writes exclusively through DB — nothing touches localStorage directly.

javascript
DB.insertOne("ruet_products", { name: "Notebook", price: 180 });
DB.findWhere("ruet_orders", o => o.userId === session.id);
DB.updateById("ruet_users", userId, { role: "vendor" });
DB.deleteById("ruet_products", productId);

Auth Layer — Auth

A singleton IIFE handling registration, login, session management, and role-based route guards. The active session is stored under ruet_session and stripped of the password field before storage.

javascript
Auth.register({ name, email, password, role });  // "user" | "vendor" | "admin"
Auth.login({ email, password });
Auth.getSession();     // returns current safe user object or null
Auth.isAdmin();        // boolean role checks
Auth.requireLogin();   // redirects to auth.html if no session
Auth.requireAdmin();   // redirects to index.html if not admin

Script Load Order

Every HTML page loads scripts in this exact dependency order:

db.js→ seed.js→ auth.js→ utils.js→ theme.js→ toast.js→ modal.js→ navbar.js→ loader.js→ ratings.js→ products.js→ cart.js→ wishlist.js→ orders.js→ [page].js
Getting Started

Up in seconds

No build tools, no server required. Pick any option below.

01
Clone the repository
bash
git clone https://github.com/your-username/ruet-online-shop.git
cd ruet-online-shop
02
Option A — Open directly
bash
open index.html    # macOS
# or double-click index.html on Windows / Linux
03
Option B — Local server (recommended)
bash — python
python -m http.server 8000
# visit http://localhost:8000
bash — node
npx serve .
# visit http://localhost:3000
04
Login with a demo account

The app auto-seeds on first load. Use any of the credentials below.

⚠ All data lives in localStorage. Clearing site data in DevTools will reset the app to its seeded state — as will Reset All Data in the Admin Panel.
